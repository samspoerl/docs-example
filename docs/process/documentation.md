How to Write Documentation

Goals
- Be clear, concise, and actionable.
- Assume a curious developer, not a beginner.

Structure
- Start with a short purpose statement.
- Use headings and lists for scanning.
- Include one small, real-world example.

Tone
- Direct and friendly.
- Avoid jargon when possible.

Content Rules
- One document per topic.
- Avoid cross-references unless necessary.
- Prefer stable guidance over tool-specific tips.

Review Checklist
- Purpose is obvious in 10 seconds.
- Examples compile or are realistic.
- No redundant sections or filler.
